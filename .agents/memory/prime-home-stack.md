## Auth
- JWT tokens; customer in `localStorage["prime_home_token"]`, admin in `localStorage["prime_home_admin_token"]`.
- `setAuthTokenGetter` from `@workspace/api-client-react` wires the customer token into generated hooks.
- Contexts (AuthContext, AdminAuthContext) use plain `apiRequest` helper (NOT generated hooks) to avoid `queryKey` required field issues with Orval-generated hook types.
- Admin route middleware: `requireAdmin`, customer: `requireAuth`, optional: `optionalAuth`.

**Why:** The Orval-generated hooks' `query` option type requires `queryKey` which the contexts were not providing, causing TS2741 errors.

## Cart
- In-memory server-side (Map keyed by userId). Resets on server restart.
- CartContext exposes: `cart`, `isLoading`, `addToCart`, `updateCartItem`, `removeFromCart`, `clearCart`.

**Why:** Simple MVP approach; upgrade to DB for production persistence.

## TypeScript Build Order
- Run `pnpm run typecheck:libs` before running artifact-level typechecks.
- DB lib (`@workspace/db`) is composite — must be built first or artifact typechecks fail with "Module has no exported member" errors.

## Seeded Data
- Admin: `admin@primehome.lb` / `admin123`
- 10 products across 6 categories, coupon `WELCOME10` (10% off, min $30)
- Seed with: `pnpm --filter @workspace/scripts run seed`

## Route Registration
- All new routers must be added to `artifacts/api-server/src/routes/index.ts`.
- API Zod schemas live in `@workspace/api-zod` (generated by Orval from OpenAPI spec).
