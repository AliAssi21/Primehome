# Prime Home LB

A full-stack luxury home products ecommerce platform targeting Lebanon customers. Includes a customer-facing storefront and a complete admin dashboard.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080, proxied at `/api`)
- `pnpm --filter @workspace/prime-home run dev` — run the storefront (port 20131, proxied at `/`)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run typecheck:libs` — build composite libs (required before artifact typechecks)
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/scripts run seed` — seed DB with sample data + admin user
- Required env: `DATABASE_URL`, `SESSION_SECRET`

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + TailwindCSS + Framer Motion + shadcn/ui + Wouter (routing)
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- Auth: JWT (jsonwebtoken + bcryptjs)
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for API contract)
- `lib/api-client-react/` — generated React Query hooks and TypeScript types
- `lib/api-zod/` — generated Zod schemas for request validation
- `lib/db/src/schema/` — Drizzle ORM schema (source of truth for DB)
- `artifacts/api-server/src/routes/` — all API route handlers
- `artifacts/prime-home/src/pages/` — all storefront pages
- `artifacts/prime-home/src/pages/admin/` — all admin pages
- `artifacts/prime-home/src/contexts/` — Auth, AdminAuth, Cart React contexts
- `artifacts/prime-home/src/lib/api.ts` — raw `apiRequest` fetch helper (used instead of generated hooks)

## Architecture decisions

- Auth uses JWT stored in `localStorage` (`prime_home_token` for customers, `prime_home_admin_token` for admins). `setAuthTokenGetter` from api-client-react wires the customer token into generated hooks automatically.
- Cart is in-memory server-side (Map keyed by userId) — simple and fast for MVP; upgrade to DB/Redis for production.
- Contexts (AuthContext, AdminAuthContext, CartContext) use plain `apiRequest` instead of the generated hooks to avoid TypeScript `queryKey` required field issues with orval-generated hooks.
- API routes use Zod schemas from `@workspace/api-zod` for request validation.
- Admin routes are gated by `requireAdmin` middleware; customer routes use `requireAuth`.
- The frontend uses Wouter for routing, with `/admin/*` routes rendered inside AdminLayout.

## Gotchas

- Always run `pnpm run typecheck:libs` before running artifact-level typechecks; the DB lib is composite and must be built first.
- The `@workspace/api-zod` Zod schemas are generated by Orval — never edit them manually.
- When adding new API routes, register them in `artifacts/api-server/src/routes/index.ts`.
- Do NOT use `console.log` in server code — use `req.log` in route handlers, `logger` elsewhere.
- Cart is in-memory; it resets when the API server restarts.
- The `SESSION_SECRET` env var is used for JWT signing.
